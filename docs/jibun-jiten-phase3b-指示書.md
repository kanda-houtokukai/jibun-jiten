## 位置づけとゴール

3本柱の2本目「ためす」と3本目「もどってくる」を10級で成立させる。
ゴール: **10級レベルのテストを受けて、点数と合否が出て、落とした字がれんしゅうに自動で戻ること**。

## 設計判断（Chat決定済み。逸脱は停止して相談 — §7-4）

### テスト形式（独自構成。本物の大問構成はコピーしない）

- 理由: 実物の問題用紙構成の精密な模倣は著作権・商標の両面で避ける。守るのは「150点満点・合格ライン120点・3分野」のみ
- 構成: **よみ10問（各5点）／かき14問（各5点）／かくすう6問（各5点）＝30問150点。合格120点**
- 名称は「10きゅうレベル テスト」等の独自表現。「漢検」の語は一切使わない（CLAUDE.md準拠）

### 解答方式

- よみ: 文を表示（対象字のみ漢字・強調）→ 読みを3択。誤答肢は他の字の選定読みから機械生成（モーラ数が近いものを優先。正解と紛らわしすぎる同音は除外）
- かき: 文を表示（対象箇所をひらがな＋空欄マス）→ 手書き（ガイドなし・matcher判定・しきい値はれんしゅうと同じ暫定値）
- かくすう: 字を表示 →「なんかくで かく？」を3択（正解±1〜2の数字を誤答肢に）
- キーボード入力は使わない（小1に無理）

### テスト中の振る舞い

- 救済なし・「おしい」表示なし。判定は静かに記録して次の問題へ（ためすは測る場）
- 時間制限なし（v1）。**中断・再開できること**（進行状態を IndexedDB に保存。小1が30問を一気にやれない前提）
- 出題は sentences-g1.json からランダム抽出。同一テスト内で同じ文・同じ字の重複を避ける

### 結果と「もどってくる」

- 結果画面: 点数／150、合格ライン到達の可否（合格時だけ祝う。不合格時は「あと○てん」と前向きに。禁止語規範は texts.js 準拠）
- 間違えた字の一覧 → **復習キューに積み、「きょうの5もじ」が復習キュー優先で出題する**よう れんしゅう側の出題ロジックを拡張
- テスト履歴の保存は最小: **最新と自己ベストのみ**（スコア・日時・間違い字）。SRS・履歴グラフはやらない

## タスク分割と停止ポイント

1. 出題生成モジュール（js/exam.js・DOM非依存）＋自己テスト（30問構成の成立／重複なし／誤答肢が正解と衝突しない／全字カバー可能性）→ 軽報告で続行可
2. テスト画面（出題→解答→静かな記録→中断再開）
3. 結果画面＋復習キュー＋れんしゅう側の出題拡張
4. ホームに「ためす」導線追加・バージョン表示 → 完了で停止・報告

## 検証（神田さんの作業。手順提示して停止）

1. 実機・通常ウィンドウでテストを1回通す（所要時間の体感も）
2. 途中でホームに戻る→再開できるか
3. わざと数問落とす→結果画面の表現と、きょうの5もじに落とした字が先頭で出るか
4. 満点時・合格時・不合格時の3パターンの文言確認（開発用に結果を作れる仕掛けは Code の裁量で可、ただし本番導線から見えないこと）

## やらないこと
