# じぶん字典 情報設計（アプリ構造）確定・2026-07-20

**重要な方針転換の記録。** これまでの実装(v0.9)は「考えた機能を1画面に縦に貼り付けただけ」で、アプリとしての階層構造がなかった。神田さんの指摘「アプリ感がない・今の見た目は30%」を受け、アプリ全体の情報設計をやり直す。デザイン言語(色・書体・世界観・定着)は生きるが、**画面の組み方は設計からやり直し**。

## 大前提：iPad横向き中心

- **iPad横向き(landscape)を中心に設計**し、スマホ(縦)にも対応。
- これまでのモック・実装は縦向きスマホ基準(max-width:420px)だった。**横長の広い画面を主戦場にレイアウトし直す。**
- 横は幅が広い＝左右に配置できる。縦スマホの「上から下へ1本の流れ」とは別のレイアウトになる。

## 背骨：学年が主・級が従の「二軸」

神田さんの核心的意図: **基本は「学年で学ぶ」。漢検はその中に内包される腕試し。** 級を前面に押すと塾っぽい。学年を前面にすると「学校の漢字を楽しく身につける」アプリになる(当初からの思想=小学校で学ぶ常用漢字を網羅)。

- **横軸=学年(主)**: 1〜6年。普段の顔。自分の学年で土台を固める
- **縦軸=級(従)**: 10〜5級。「もっと上に挑戦」の伸びしろ。普段は控えめ、でも道は最初から用意
- 2年生の子が、自分の学年(9級相当)の腕試しをしつつ、その気になれば上級にも挑戦できる

## 画面地図

- **ホーム(玄関)**: 「○ねんせいの かんじ」が主役。今日やること＋自分の学年の到達状況。始める場所に徹し、機能そのものは置かない
- **学年でまなぶ(背骨)**: 1〜6年。基本は自分の学年。各学年に入ると2つの道:
  - **れんしゅう**: その学年の字を**分野別**に(よみ・かき・筆順…学年が上がると部首・送り仮名等が増える)。「かきだけ集中」「筆順だけ集中」ができる
  - **腕試し(漢検内包)**: まず自分の学年相当の級の模擬テスト。加えて**分野別テスト**(よみだけ30問等)も選べる
- **もっと上に挑戦(縦の伸びしろ)**: 自分の学年を超えて上の級へ。9級→8級→…。漢検の級体系が縦に繋がる。普段は控えめ
- **じぶん字典**: 学年をまたいで書いた全字の作品集。全体の資産として独立

## 拡張への耐性（この構造の狙い）

級が増えても学年の棚に、上級挑戦は縦ラインに、分野が増えても各学年のれんしゅうの中に——**置き場所が最初から決まっている**。「拡張を踏まえたいい画面」への答え。模擬テストの分野別集中(項目ごとだけの練習)もこの構造に収まる。

## 表記ルール（ふりがな）：案A＋案C併用・段階設計

UI文字を3種に分け、**種類1だけ**学年で変える。混同しないこと。

- **種類1=UIの言葉**(ボタン名・メニュー・「きょうの5もじ」等): **学年で漢字度を変える**←本件の対象
- **種類2=問題文の中の文字**("いもうとが生まれた。"): **触らない**。対象字のみ漢字・他ひらがなの現ルール維持(表記でなく出題設計の問題)
- **種類3=学ぶ対象の漢字**(手本・じぶん字典の字): 全学年で漢字(KanjiVG)。表記議論の外

### 種類1の基準（確定）
- **選んだ学年までに習う字は漢字で書く(案A)＋その漢字にふりがなルビを振る(案C)**。それより上の学年の字はひらがな
- **段階設計**: 低学年(1〜2年)はひらがな中心・ルビ付き漢字はごく一部 / 中学年(3年〜)で漢字＋ルビが本格化。学年が上がるほど漢字度が増す
- 4年生の例: 1〜4年の字は漢字＋ルビ、5〜6年の字はひらがな
- これで「全ひらがな→6年生に幼稚」も「習いたての字は読めないかも」も両方解消

### ルビを消すオプション
- **将来の拡張**(6年生の「ルビいらない」対応)。まずは常時ルビで作る

### 学年の選ばせ方（確定）
- **最初に1度選んで設定に保存**(あとで変更可)。この学年設定が表記切替の基準

### 技術的裏付け
- 切替は**既存の学年別データ(1026字の学年分類)で機械判定**できる。UI文言を手作業で振り分ける必要なし。ルビもKANJIDIC2由来の読みから引ける。データ基盤がそのまま効く

## デザイン言語との関係

- jibun-jiten-デザイン言語.md の色・書体・世界観・署名演出「定着」は**そのまま生きる**
- 変わるのは**レイアウト/画面構造**(縦ベタ貼り→横向き中心の階層アプリ)
- ホームモック(home-mock.html)は縦スマホ版として参考に残すが、**iPad横向きで作り直す**
