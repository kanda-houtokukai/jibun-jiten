# じぶん字典 デザイン言語（確定・2026-07-18）

ホームモックで神田さんが方向を承認。**全画面はこれに従う。** これがUIの正本。

## 世界観（一言）

「にぎやかなゲーム」ではなく「**自分だけの静かな道場／作品集**」。
キャラクターなし・ゲーム的演出なし。主役は常に**子ども自身が書いた字**。装飾はそれを引き立てる脇役に徹する。

## 却下した方向（戻らないための記録）

- キャラクター・マスコット → なし（独自性が死ぬ・本人が明確に不要と表明）
- ゲーム的な賑やかさ（別途開発中のプログラミングゲーム的なノリ）→ 求めていない
- 和風・習字・生成り紙に墨色 → **却下**。既存漢字アプリが全部やっており独自性にならない
- クリーム背景(#F4F1EA)＋セリフ＋テラコッタ(#D97757) → **意図的に回避**。AI生成デザインの定番＝「AIが作った印」になる

## カラートークン

| 名前 | hex | 役割 |
|---|---|---|
| paper | #F7F8FA | 背景。生成りでなく「かすかに青い冷たい白＝静かな机」 |
| paper-edge | #EDEFF3 | カードのふち・かすかな段差 |
| ink | #1B2A4A | **主役の字（子どもの筆跡）**。黒でなく万年筆の青黒 |
| ink-soft | #3A4B6E | インクの弱め（ラベル等） |
| ghost | #C7CCD4 | **手本(KanjiVG)・未習得**の脇役グレー。主役を邪魔しない |
| ghost-faint | #DDE1E7 | さらに淡い |
| grow | #3A9D6B | **達成・育つの新緑の一点**。金や赤でなく「育つ」の象徴。うるさくない |
| grow-bg | #E8F3ED | 新緑の淡い地 |

- テラコッタ・クリームは使わない（AI臭回避の肝）
- 色は増やさない。この8色で全画面。達成は新緑一点に集約

## 書体

- UI文言（ひらがな）: 丸すぎず硬すぎない知的なゴシック（Hiragino Sans系）。**まるもじに逃げない**（安っぽくなる）
- 数字（点数・画数）: 性格のある明朝系。「150」が作品のように見える扱い
- 漢字: 全部 KanjiVG（ベクター）→ 書体問題から自由。ここは既に勝っている
- 太さは2種（400/600）まで。重くしない

## 署名要素（このアプリが記憶される、たった一つ）

**「定着（てい）」の演出。** そらがきで1字を書き上げた瞬間、線が濃紺のインクとして紙に染み込んで定着する一瞬（約0.4秒）。ファンファーレ・星・キャラの反応は出さない。「自分の線が作品になった」手触りだけを残す。
- 理由: 賑やかな演出は3日で飽きる。「自分の線が美しく定着する気持ちよさ」は飽きない。親が"すごい"と言うのは点数でもキャラでもなく、わが子の手の字がきれいに飾られた瞬間
- じぶん字典に並ぶのは、この定着した一字一字
- 大胆さはここ一箇所に集中。他画面は静かに保つ（frontend-designスキル「spend your boldness in one place」）

## レイアウト原則

- ホームの主役は**メニューボタンではなく「これまで書いた字の壁」**。開いた瞬間「自分の場所だ」と感じさせる
- 濃紺インクの塊は画面に1つだけ置き、そこに新緑がにじむ（＝定着の予告）
- 1画面1目的・大きなタップ領域・応答速度優先。アニメは短く、やりすぎない
- 文言は全ひらがな・禁止語（×・ざんねん・まちがい・しっぱい）継続・texts.js集約

## モック

- home-mock.html（ホーム・承認済み）。本番はフォント代用部分がKanjiVG＋本人筆跡になり、質感はさらに上がる

## 次に作る画面（未着手）

- じぶん字典（自分の字が作品として並ぶ）

# 追記・2026-07-20：UIの質感「積み木」確定

情報設計のやり直し(縦ベタ貼り→学年二軸の階層アプリ)に伴い、レイアウトの質感を探索。3案比較→立体感2パターン→段差の向き・量、と実物モックで絞り込み、神田さんが確定。

## 確定した質感：積み木・知育玩具の洗練

- 方向は「子ども寄りの洗練」。大人向けSaaSのミニマリズム(=AIが作りがちな冷たい端正さ)は**却下**。原色ポップ・キャラの子どもっぽさも却下。狙いは「上質な知育玩具・積み木」の中間地点
- **大きな色ブロックを敷き詰めた構成**。各機能が独立したブロックとして置かれる
- **右下に斜め3pxの控えめな段差**(box-shadow: 3px 3px 0 [濃い同色], 5px 6px 12px rgba(51,50,46,.10))。光が左上から差す自然な陰影。真下でも強い厚みでもなく「ごく少し」。触りたくなる物理感を、押しつけがましくなく
- 角丸は大きめ(22px)。ブロックらしいふっくらした角

## 積み木版のカラー(ホーム3年生モタックより)

- ⚠️注意: この積み木版パレットと、既存デザイン言語の「冷たい白／濃紺インク」パレットは**別物**。今回のUI刷新で温かいオフホワイト＋カラーブロックに寄せた。ただし**署名演出「定着」の濃紺インク(#1B2A4A)と、じぶん字典で字を見せる時の主役性は維持**する（字そのものは濃紺のまま・ブロックは器）。この2つの世界観の接続は、じぶん字典と練習画面のモック時に詰める（宿題）

# 追記・2026-07-20（2）：じぶん字典の形と「器×字」の接続確定

じぶん字典モック2案比較→案2確定。宿題だった「積み木の温かい器」と「濃紺インクの字」の接続に答えが出た。

## 確定：じぶん字典＝学年の区画に分けた図鑑（案2）

- **学年ごとに区画を分ける**。左に学年タグ(1年=10級…と級を併記)、右にその学年の字。二軸構造(学年主・級従)を字典でも体現

# 追記・2026-07-21：ホームのレイアウト確定＝当初Y案（みち・イラスト非依存）

## 経緯（長い探索の記録）
ホーム質感（積み木）確定後、レイアウトが未決と判明。研究ベースで3案提示（X=ミッション型/Y=みち型/Z=折衷）→ 神田さんはYとZで迷い、Yを選ぶなら汽車が線路を進む等もっと豊かにできるはずと提起 → 汽車入りY+を作成 → **汽車のイラストがイマイチ**（手描きSVGの限界。図形の組合せでは記号的な絵にしかならない）→ イラスト調達手段を確認 → **このチャット内でClaudeが画像を出す手段は事実上手描きSVGのみ**（画像生成ツールは手元になく、lucide等のアイコンライブラリは実装時にCodeが使うものでモックでは手SVG代用）と判明 → よってイラストの質に成否が依存する路線はモックで判断しきれない → **汽車Y+を破棄し、当初のY案で確定**。

## 確定：ホーム＝当初Y案「みち型・イラスト非依存」
- 上帯に学年ピル
- 1年→6年の駅が**横一列**に並ぶ（色ブロック＋文字のみ）。済学年=緑＋チェック、現在学年=大きく膨らみ中に「きょうのぶん →」、未到達=淡い破線
- **「もっと上に挑戦」は別枠でなく道の先（4〜6年＝7〜5級）として見える**＝学年主・級従の二軸を体現
- 下部に練習・腕だめし・じてんの3ドック
- **乗り物・キャラのイラストは置かない**。駅=色ブロックと文字で「みち」を成立させる（イラスト依存を避ける）
- 質感は確定版（積み木・右下3px段差・確定パレット）
- 確定モック: ホーム-レイアウト3案.html の案Y

## ★重要な教訓（次プロジェクトにも効く・design-processスキルにも反映すべき）
1. **手描きSVGはモックの「方向を握る下書き」であり、最終イラストの品質は出せない**（図形組合せは記号的な絵が限界）
2. **このチャット環境でClaudeが絵を出す手段は事実上手SVGのみ**（画像生成は手元になく、アイコンライブラリ=lucide等は実装時にCodeが使う）。最終イラストは実装時に①lucideアイコン②AI画像生成（Code側/別環境）③商用可の既製素材、で質を上げる別工程
3. 帰結: **モック段階では「イラストの巧拙に成否が依存する構成」を避け、色ブロック・文字組み・段差など"コードで確実に質が出る領域"で成立する構成を選ぶ**。凝ったイラストが要る世界観は、モックで良し悪しを判断しきれない

# 追記・2026-07-22：練習の分野選択 確定（案B-2「めもり」）

## 解いた設計問題：分野は学年で3→11個に増える
分野をそのまま並べる設計は高学年で破綻する（1年=よみ・かき・筆順の3個 → 6年=四字熟語・誤字訂正等11個以上）。
子ども向けUX研究は「1画面の選択肢は3〜5個」「1画面1目的」「要素過多は混乱を生む」で一致。よって分野の直接列挙は不可。

## 確定：分野を「4つの入口」に束ねる（入口の数は学年で変わらない）
- **よむ**（よみ・同音異字・同訓異字…）／**かく**（書き・送りがな・誤字訂正…）
- **かたち**（筆順・画数・部首・熟語の組み立て…）／**ことば**（対義語・類義語・四字熟語…）
- 新分野は既存の入口の中のチップとして増えるだけ。**6年11分野でも入口は4つのまま**＝3〜5択の原則を6年まで守れる
- 1年生は「ことば」が未開放で3つだけ表示 → 学年が上がると入口が開く（成長が構造に現れる）
- ⚠️未確定: 「熟語の組み立て」を"かたち"に置いたが"ことば"寄りの可能性。神田さんの漢字教育の感覚で後日調整可

## 確定：にがて導き帯を置く（案B）
- 画面上部に一本だけ「いま にがてなのは ◯◯・のこり◯もん → ここからやる」
- 根拠: 研究の「選択肢だけでは不十分、子どもには軌道に乗せる導きが要る」。かつDuolingoの反発（強制一本道は自律性を殺す）を避け、**迷う子は押すだけ／選びたい子は下から選べる**両立

## 確定：視覚の作り込み（案B-2「めもり」）
神田さん指摘（余白が狭い／ブロック内の余白が大きすぎる／進捗バーが単純すぎ）を受けた改訂。
- **外周余白を拡大**（24px/32px）、ブロック間 20px
- **ブロック内の空洞を"意味"で埋める**: space-between をやめ、一行説明（「よみかたを こたえる」等）を追加。装飾でなく子どもの理解を助ける情報で埋める
- **進捗＝めもり（物差し）**: 目盛りつきトラック＋つまみが現在位置を示す。文房具の精度感。※積み木型（積み上がる段）も作ったが不採用
- **数字は明朝**（デザイン言語の「数字は性格のある書体」を適用。従来は適用漏れ）
- アイコンを台座（角丸プレート）に載せる／ブロック上辺に淡い光沢＝積み木の面の反射／にがて帯に星マークと厚み
- 進捗ラベルは「のこり◯もん」等の残量提示（前進を促す）。他人との比較は一切なし

# 追記・2026-07-22（2）：腕だめし 確定（案1「本番が主役」）

## 調査：試験不安（この画面固有の急所）
- 高stakesテストは低stakes（教室テスト）より有意に不安が高い。認知面・身体面の症状も強い（小3〜5対象の研究）
- **アジアの子どもは他地域より試験不安が高い**（親の期待・高stakes入試文化）。日本の子ども向けは特に配慮が要る
- 低stakesの短い挑戦の反復は不安を下げる。逆に高stakesの繰り返しはスタミナでなく燃え尽きを生む
- 最大の危険：失敗の恐れから子どもが低い点を「自分の頭の悪さ」と内面化すること
- → 神田さんが当初から求めた「項目ごとに集中して練習」は、**不安対策としても正しい要求**だった

## 確定：案1「本番が主役」
- **もぎテスト（30もん・150点・約15分）を上部に大きく**、墨色 #33322E で別格に。4色の分野別に対し墨＝「特別なもの」の格を色で表現
- 下に**短い分野別テスト4つ**（よむ10もん3分／かく10もん5分／かたち10もん3分／ことば8もん3分）
- **分野別は練習画面と完全に同じ4色・同じ言葉**（研究「同じ操作には同じ色と形を」。子どもは"よむ=緑"を一度覚えれば練習でもテストでも迷わない）

### 案2（短い分野別が主役）を採らなかった理由
Claudeは不安配慮から案2を推奨したが、神田さんが案1を選択。理はこちらにある——**このアプリの目標は最初から「漢検の合格ライン」という本物**。そこを小さく扱うと目標の存在感が薄れ「なんとなく練習するアプリ」になる。本番を堂々と主役に置くことで目指すものがはっきりする＝当初の設計思想（人工的ポイントでなく本物の基準を目標に）と整合。

### 不安への配慮は「主役の大きさ」でなく仕掛けで担保する
- **「なんかいでも うけられるよ」を画面に明示**（失敗が取り返しのつかないものでないと保証する）
- 短い分野別テストを常設（低stakesの受け皿）
- **合格ラインを目盛りに描く**（自己ベストのつまみ＋120点の黄色い線。あと何点で届くかが視覚で分かる＝点数を人格でなく"伸び"として見せる）
- 「まちがえた 字は あとで れんしゅうに もどってくるよ」を事前に伝える（間違い＝罰でなく次の学びへ）
- 前回・自己ベストの併記。他人との比較は一切なし

## 確定モック
腕だめし-2案.html の案1
