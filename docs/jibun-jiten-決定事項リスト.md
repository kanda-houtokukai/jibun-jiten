## 1. 案件の基本

- 種別: **一般公開する個人プロダクト**（法人PTではない。pt-context は適用しない）
- 対象: 通常学級の小学1〜6年生。配慮版（UDフォント・読み上げ等）は将来のパーキング
- 成果物: スマホ・タブレットのブラウザで動く漢字学習アプリ（後日PWA化）
- 収益化: **完全無料・広告なし**
  - 理由: ①子どもの端末で広告ネットワークを回さない ②KanjiVGのBY-SAによりデータ囲い込み型の有料化が成立しにくい ③商標・ライセンス両面のリスク軽減

## 2. プロダクトの芯（署名コンセプト）

- **「書いた字を捨てない」**。判定に使った筆跡ベクターをそのまま資産として保存する
  - **じぶん字典**: 1026字マップが、フォントではなく本人の字で埋まっていく
  - **成長リプレイ**: 同じ字の過去→現在の変化を再生（親向けの体験の核）
  - **にがて印刷**: 間違えた字だけの練習プリントをPDF生成（親モード・後期フェーズ）
- 筆跡の保存は1字につき「はじめて・ベスト・さいきん」の**3枚まで**（容量が増え続けない）
- 意欲設計はごほうびを外付けしない。目標は級と合格ラインという「本物」を使う
- 失敗表現は「×」でなく「おしい！3画目をもう一回」。他人との比較機能は作らない
- **やらないこと（確定）**: ガチャ / ペン・スキン解放 / ランキング / キャラ育成 / 広告 / 課金 / アカウント登録 / SNS共有
- 品質の柱 = **4ゼロ**: 広告ゼロ・課金ゼロ・登録ゼロ・通信ゼロ（オフライン動作）

## 3. 手書きまわり

- 方式は**照合型**（模範ベクターとの比較）。一般の手書き認識（ML Kit等）は使わない
  - 理由: 答えが既知 / 1画ごとの理由つきフィードバックができる / ML Kitは筆順を見ず、Web非対応
- 判定哲学: **筆順・画数は厳格、字形・とめはねは寛容**（漢検の採点思想に合わせる。採点は常用漢字表準拠・くずし字や乱雑な字は対象外だが、とめはねの過剰減点はしない）
- フィードバックは1画ごと即時。KanjiVGの画種（kvg:type）と部品分解を子どもの言葉に翻訳して指摘する
- **部品ビュー**: 新出字で既知の部品が光る（KanjiVGの分解データを使用。8級からの部首問題への橋）

## 4. 学習の骨格（3本柱・これ以外は作らない）

1. **おぼえる** — 1字ずつ: 読み → 筆順アニメ → なぞり書き → 空書き
2. **ためす** — 漢検形式の模擬テスト（級ごと・満点/合格ライン再現）
3. **もどってくる** — 間違えた字が自動で復習に戻る

- 1回 = **5字・3分**（低学年の集中力に合わせた最小単位）

## 6. データ設計

- **3層＋関係**に分ける。1字1レコードにしない
  - 字テーブル（1026行）: char / grade / 総画数 / 部首 / 読み{音,訓} / strokes[]
  - 語テーブル（数千行）: word / 読み / 使用漢字[] / 語種 / 熟語の構成型 / 意味
  - 文テーブル（数千行）: 本文 / 空欄位置 / 答え / 誤字訂正用の誤字
  - 関係テーブル: 対義語ペア / 類義語ペア
- **級は振らず計算で出す**: 語の学年 = 構成漢字のうち最大の学年。人力の級付けゼロ
- 例文・熟語は自前生成（既存問題集からの流用は著作権侵害）。数値・語義には⚠️要検証を付けて運用（鉄則5）
- 概算容量: KanjiVG由来ストローク約1.5MB＋語・文で数MB → 全データ端末内に載る見込み ⚠️Phase 1で実測

## 7. 技術方針

- **Web（PWA）先行**。ネイティブ化は後日Capacitorで可能（逆は不可）
- ホスティング: **GitHub Pages**（完全静的・無料・商用可。Vercel Hobbyは商用不可注意が外部サービス台帳にあり）
- 保存: **IndexedDB ＋ JSON書き出しバックアップ**（勤務表ツールの実績構成の発展形。localStorageから変更した理由: 筆跡データの容量）
- 筆順データ: **KanjiVG（CC BY-SA 3.0）**。派生データは同ライセンスで公開しクレジット表記 ⚠️公開前にライセンス原文を最終確認（Claudeは弁護士ではない）
- 手書き入力: Pointer Events ＋ canvas `desynchronized`。Safariの制約（getCoalescedEvents未対応・Scribble干渉）はPhase 0で実測判定
