# jibun-jiten Phase 3d 指示書（連濁バグの一括修正と再発防止）

対象: Claude Code / 推奨モデル: Fable 5（日本語の連濁の網羅的判定＝言語知識が本体）
前提: 台帳・決定事項リストを読むこと。Phase 3c 検収済み（v0.5）。

---

## 発覚した不具合（背景）

よみ問題「けん玉であそぶ」で対象字 玉 の正解を「たま」としているが、この文脈での実際の読みは連濁で「だま」。3択の正解と文中の実読みが食い違う。
根本原因: Phase 3a の文生成・機械チェックに連濁（前の語につられ語頭が濁音/半濁音化する現象）の概念がない。「登録読みに含まれるか」しか見ないため、濁点付きで読む文脈をすり抜ける。**1件ではなく同型の穴が他にもある前提で洗う**。
重要: この種は形式的機械チェックでは検出不能（意味と音を人/言語知識で見て初めて分かる）。後回しにした照読が拾うはずだった類。

## タスク1: 全164文の連濁・音変化スキャン

- `data/sentences-g1.json` の全文について、対象字が文脈中で登録読みと違う音になっていないかを判定する
- 主に連濁（例: 玉→だま、川→がわ、花→ばな、人→びと、時→どき 等）。加えて促音化・その他の音便で登録読みと表層がずれる文も拾う
- 出力: `data/rendaku-audit-g1.json`（該当文の id・字・登録読み・文脈での実読み・修正方針を一覧）＋確認ページに表示
- ⚠️川の「天の川＝あまのがわ」は既存文に存在の可能性（Phase 3a報告に「天のがわ」あり。target.reading が「あま-」等になっていないか要確認）

## タスク2: 修正（正本スクリプト経由）

各該当文について、次のいずれかを選ぶ（字ごとに自然な方を）:

- **(A)** その字が連濁しない別の文脈に文を差し替える（玉が語頭or単独で"たま"になる文へ）
- **(B)** 文はそのままに、出題読み・正解を実読み（例「だま」）に合わせる。ただしその濁音読みが exam-readings-g1.json に無い場合は、連濁は派生なので**原則Aを優先**（濁音を正式な出題読みとして登録すると、単独字の出題で「玉=だま」を正解にしかねない副作用があるため）
- 判断に迷う字は audit に残して停止時に報告（Chatで方針決定）
- 修正は tools/build_exam_g1.py の正本を直して再生成（Phase 3a方式）。手でJSONを書かない

## タスク3: 再発防止（スキーマ/チェックの強化・設計判断あり）

以下を実装し、方針を報告:

- 文レコードに表層読みの検証を組み込む: target.reading は「その文脈で実際にそう発音する読み」であることを不変条件にする（登録読みに含まれることに加え、連濁時は濁音表層を明示的に許容 or 除外のどちらの設計を採ったか報告）
- build スクリプトに連濁疑い検出の警告を追加（対象字の直前に語がある＝語頭でない位置で、清音読みを出題している文にフラグ）。完全自動判定は不要だが、将来 小2以降を足すとき同じ穴に落ちないための警告装置を残す
- この設計判断は手戻り防止の要。判断が割れたら停止して相談（§7-4）
