## タスク1: プロトタイプ実装

成果物: index.html **1枚**（ライブラリなし・素のJS）

### 完了条件
- [ ] 画面中央に大きな正方形のマス1個（十字の薄いガイド線つき）
- [ ] 指・スタイラスの両方で書ける（Pointer Events。マウスでも動くと開発が楽）
- [ ] 「1画もどす」「ぜんぶ消す」ボタン
- [ ] 計測表示: 現在ストロークの点数 / pointermove間隔からの推定Hz / **バージョン番号**
- [ ] 何も保存しない・判定しない・漢字データなし

### 技術要件（落とし穴込み）
- canvas は `getContext('2d', { desynchronized: true })`
- マス要素に `touch-action: none`（これがないと描画がスクロールに食われる）
- 線は座標を結ぶだけの単純描画でよい（スムージング・筆圧は今回やらない。**生の書き味で合否を見るため**）
- Safari は getCoalescedEvents 未対応 → 使わない。pointermove の素点で描く
- 全画面キャンバスにしない。マス内のみ描画（iOS Safari の大キャンバス性能劣化を避ける）
- ページ内にバージョン表示（例: v0.1）。GitHub Pages の反映確認用（★過去案件の教訓: push後1〜2分反映・シークレットウィンドウで確認）

### やらないこと
判定 / 保存 / 漢字表示 / 装飾 / PWA / 複数マス / スムージング調整
