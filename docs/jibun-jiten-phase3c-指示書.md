# jibun-jiten Phase 3c 指示書（「なんでかな?」ふりかえり表示）

対象: Claude Code / 推奨モデル: Fable 5（子どもの感情に触れるUX設計。文言と見せ方の質が本体）
前提: 台帳・決定事項リストを読むこと。Phase 3b 検収済み（v0.4）。

---

## なぜ作るか（背景・設計者の意図）

現状の結果画面は「どの字を落としたか」で止まり、「なぜ×か」を子どもに見せない。
子どもは必ず「ちゃんと書けたのに」「なんで?」となる。ここを埋める。
朗報: **必要な情報は既にある**。matcher は判定時に perStroke（画ごとの score/dist/dirOk）を返す。
結果画面がこれを捨てているだけなので、拾い直して「見せる」。新しい判定ロジックは作らない。

## 最重要の設計思想（これを外すと逆効果になる — 必ず守る）

この機能は諸刃。対処すべき「なぜ」は2種類あり、扱いが違う。

- **種類1（正しく捕捉した間違い）**: 筆順が逆・画が足りない/多い等。matcher が該当画を特定済み。→ 重ねて見せて、子ども自身に気づかせる
- **種類2（誤判定＝本当は正しいのに×）**: しきい値の誤判定。→ ここに饒舌な説明を出すと「機械がおかしい」を強化してしまう。**説明で押さない**

したがって:

1. 機械は断定・講釈しない。「3画目の形が誤っています」のような上から目線の説明文を出さない
2. 見せ方は視覚が主役（自分の筆跡と手本の重ね）。言葉は最小限の中立な補助のみ
3. **逃げ道を必ず作る**: どの振り返り画面にも「もういちど かく」。納得できなくても書き直して「やっぱり書ける」を確認できる。これが「ちゃんと書けたのに」への実質的な答え
4. 禁止語規範（×・ざんねん・まちがい・しっぱい）は継続。全ひらがな・texts.js集約

## タスク1: 結果画面に「なんでかな?」入口

- 現在の結果画面の間違い字リストの各字に「なんでかな?」を付ける（タップで振り返りビュー）
- かき問題で落とした字だけが対象（よみ・かくすうは3択なので「重ねて見せる」対象外。これらは種類2的な説明をせず、正解を示すだけにとどめる → タスク3）

## タスク2: 振り返りビュー（かき用・この機能の本体）

- 自分のその時の筆跡（テスト中に書いたストローク）と**手本(KanjiVG)**を同じマスに重ねて表示
  - ⚠️前提確認: テスト中のかき筆跡が保存されているか。なければ、結果からの振り返りのためにテスト中のかき筆跡を控える最小の保存を足す（Phase 2 のじぶん字典3枚ルールとは別。テスト1回分の一時保持でよく、次のテスト開始で破棄してよい）。既存の筆跡ストア(v2)は壊さない・非破壊マイグレーションで
- matcher の perStroke を使い、問題の画だけを色分け（手本=基準色・自分=別色、問題画を強調）
- 画種別の見せ方（種類1の中の下位分類。過剰にならない範囲で）:
  - 筆順違い/向き違い（dirOk=false）: その画を順番にアニメして「こっちのむき」を視覚で
  - 画数過不足: 「かくが たりない/おおい」の一言＋手本の画数アニメ
  - 形ずれ(dist大)のみ: 重ね表示だけ（言葉での指摘はしない。種類2の可能性が高い領域なので断定を避ける）
- 画面内に必ず「もういちど かく」→ その場で書き直し→ 書けたら軽く「できたね」（テストの点は変えない。あくまで納得のための再挑戦）
- 文言は中立・短く・ひらがな。例（texts.jsへ）: 「ここを みてみよう」「こっちのじゅんばん」「もういちど かく」「できたね」。「まちがい」系・説明口調は使わない

## タスク3: よみ・かくすうの振り返り（軽く）

- 3択で落とした問題は、振り返りでは正解を示すだけ（「こたえは ○○」）。理由の講釈はしない
- かき筆跡のような重ね表示は不要

## タスク4: 仕上げ

- バージョン表示更新
- 開発用に、特定の誤り方（筆順違い・画数不足・形ずれ）を含む結果を再現できる手段があると検証しやすい（Code裁量・本番導線から不可視・保存しないこと）

## 停止ポイント

タスク1〜4完了で停止・報告（テスト中かき筆跡の保存有無と採った対処・振り返りビューのサンプル説明）。

## 検証（神田さんの作業。手順提示して停止）

1. 通常ウィンドウでテストを1回、かきをわざと筆順違い・画数不足・雑な形で落とす
2. 結果→「なんでかな?」→ 自分の字と手本が重なって見えるか・問題の画が分かるか
3. 「もういちど かく」で書き直せるか（ここが感情の逃げ道として働くか）
4. 説明が「上から目線・くどい」と感じないか（感じたら要調整。この機能はやりすぎが失敗）
5. 可能なら子どもの反応を見る（「なんで」が和らぐか。無理にとは言わない）

## やらないこと

matcher の判定ロジック変更（しきい値調整は別件・子ども試用後）/ よみかくすうへの理由説明 / 音声 / 履歴への振り返り保存 / 親モード / 9級以降
