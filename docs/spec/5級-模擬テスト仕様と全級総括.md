## ★★最重要の発見：誤字訂正が出題されていない

公式の出題内容表は5級に**誤字訂正**を挙げているが、**この回（2024年度第3回）には誤字訂正の大問が存在しない**。

これは協会の注記「**毎回すべての分野から出題されるとは限りません。また、このほかの分野から出題されることもあります**」の実例。

### 設計への帰結
**出題分野は「毎回出るリスト」ではなく「出うるリスト」。1回分の過去問で構成を固定すると実際とズレる。**
→ 級ごとの構成は**「1つの固定パターン」でなく「出題タイプのプールと、そこからの選び方」**として持つべき。
→ データファイル化の判断が、ここで最終的に正当化された。

実装案:
```json
{ "level": 5, "totalPoints": 200, "passLine": 140,
  "observedPattern": [ /* 2024年度第3回の実構成 */ ],
  "typePool": [ "yomi_write","radical_choice","stroke_and_count","okurigana",
                "onkun_combo","yoji_jukugo","antonym_synonym","jukugo_from_meaning",
                "jukugo_structure","doon_dokun_write","kanji_write","goji_teisei" ] }
```
- 既定は観測パターンで出題し、`goji_teisei`（誤字訂正）のようにプールにはあるが観測回に出なかったタイプも実装しておく
- 将来「たまに出る分野」として混ぜる、あるいは練習側の分野別で扱う

## ★部首の問い方が級で変わる
| 級 | 部首の形式 |
|---|---|
| 9級 | なかまの漢字（部首記号のみ提示） |
| 8級 | なかまの漢字（**部首名**つき: こざとへん等） |
| 7級 | 部首のなかまの漢字 |
| 6級 | **部首名は記号選択＋部首を記述** |
| 5級 | **部首・部首名とも記号選択** |
段階的に易しく／難しくなるのでなく、**形式そのものが違う**。→ 部首データには「部首記号・部首名・所属漢字」の3点が必要。

## 設計に効く結論

1. **構成は級ごとに大きく異なり、単調増加ではない**（81→105→100→120→120→120問）。**データファイル化は必須**
2. **出題タイプは現れたり消えたりする**。積み上げでなく入れ替わり。→ 級定義は「使うtypeの集合」で持つ
3. **同じ分野でも級で形式が変わる**（部首は5段階で別形式、筆順は7級から総画数と複合）
4. **公式の出題内容表と実物にはズレがある**。表になくても出る（10級の対のことば、9級の部首）／表にあっても出ない回がある（5級の誤字訂正）
5. **記号選択は悪ではない**。7級以上は30問前後が記号選択。ただし**誤答肢が紛らわしい**（実ケン→健/競/験）。10級で読みを3択にしたのが誤りだったのは選択肢が易しすぎたから
6. **自動生成できる出題が相当ある**（筆順・総画数・音訓判別・音訓組合せ）。KanjiVG＋KANJIDIC2で機械生成可能。執筆コストを大きく下げられる
7. **部首データ（漢検要覧準拠）が最大のブロッカー**。9級から5級まで一貫して出題され、6級では部首そのものを書かせる。**優先度を上げて解決すべき**
