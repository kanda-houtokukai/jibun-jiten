# jibun-jiten Phase 2 指示書（おぼえる＋じぶん字典）

対象: Claude Code / 推奨モデル: Fable 5（子ども向けUIの質と、手戻りの痛い保存スキーマ設計を含むため）
前提: 台帳・決定事項リストを読むこと。matcher v1 は合格・しきい値は初期値で暫定確定（子どもの実書きでの再調整余地あり）。

---

## 位置づけ

道具（viewer / match-test）から、**初めて子どもが使う製品画面**を作るフェーズ。
署名機能「じぶん字典」がここで初めて形になる。範囲は10級80字のまま。

## 設計判断（Chat 決定済み。逸脱が必要なら停止して相談 — §7-4）

### 画面構成（3枚だけ）

1. **ホーム**: 「きょうの5もじ」ボタン＋すすみ具合（80字中いくつ）＋じぶんじてんへの入口
2. **れんしゅう**: 1字フロー（下記）を5字ぶん回して終わる。1回=5字・3分の設計どおり
3. **じぶんじてん**: 80字マップ

### れんしゅうの1字フロー

①よみ＋筆順アニメ（KanjiVG）→ ②なぞり（薄いガイドの上に書く・matcher判定）→ ③そらがき（ガイドなし・matcher判定）

- 「おしい」のときは該当画だけアニメで見せて再挑戦
- 3回つまずいたら、ガイドを薄く出して成功で締める（失敗で終わらせない。ここは意欲設計の要）

### 文言・トーン

- 全文言ひらがな（小1が読める）。禁止語: ×・ざんねん・まちがい・しっぱい。使う語: 「おしい！」「もういっかい」「できた！」
- 文言は1ファイル（js/texts.js 等）に集約。理由: 後で学年別に調整するため
- 見た目: 1画面1目的・大きなタップ領域・色は最小（背景明色＋アクセント1色＋成功色）。装飾より応答速度。アニメは短く

### 字形の扱い（重要）

- 手本・なぞりガイド・じぶん字典の模範表示は必ず KanjiVG 描画。フォント表示は字形がズレる（教科書体問題）ので手本に使わない。UIテキストはシステムフォントでよい

### 保存（js/store.js・DOM非依存・IndexedDB）

- 1字につき `{ first, best, latest }` の3枚ルール。各= `{ strokes(正規化座標列), score, at }`
- 保存が起きるのは③そらがき合格時のみ。なぞりは保存しない。理由: じぶん字典に載る=「自力で書けた」の証明であること
- best 判定は全画平均スコア。first は最初の合格作を以後不変。latest は合格のたび更新
- schemaVersion を持つ。既存筆跡データを壊しうる変更は停止して確認（CLAUDE.md 準拠）
- JSON 書き出し/読み込み（バックアップ）を設定隅に最小実装

### じぶんじてん

- 80マスグリッド。未習得=グレーの小さな字、習得済み=本人のベスト筆跡を描画
- タップで詳細: はじめて/ベスト/さいきん の3枚並び＋筆跡の順再生（成長リプレイの最小版）

### URL の一等地

- 子ども向けアプリを index.html に据える。現在の書き味テストは test-draw.html に改名して残す（viewer / match-test も開発用に残す）。理由: 公開URLの入口は製品であるべき

## タスク分割と停止ポイント

1. store.js＋自己テスト（3枚ルール: first不変・best更新条件・latest更新、export/import 往復一致）→ 軽く報告のみ、続行可
2. れんしゅうフロー（matcher 統合込み）
3. じぶんじてん（マップ・詳細・リプレイ）
4. ホーム＋導線＋バージョン表示 → 完了で停止・報告

## 検証（神田さんの作業。手順提示して停止）

3. じぶんじてんに自分の字が載るか・リプレイが動くか
