## 前セッションで判明した重要事項（引き継ぎの本体）

### ★1. matcher はかなでも動く（実機検証済み・作り直し不要）
- KanjiVG は ひらがな・**小書き（ょ・っ）・濁点付き（が=5画、濁点も画として収録）・半濁点（ぱ=4画）**をすべて収録
- matcher のパーサで解析・自己一致 OK を確認済み
- ⚠️**唯一の欠陥**: 半濁点の「゜」のような**閉じた画は始点≒終点になり方向スコアが0**→自己一致でも65点止まり。**かな照合の実装時に閉ループ画の方向判定だけ軽微な調整が必要**（matcher本体の変更はこの1点のみ）
- 小書き「ょ/よ」の相対サイズ判定（仕様の案A）は**入力マス側の前処理で実装でき、matcher非改変で足りる**

### ★2. 雨の筆順バグ — 原因を特定済み
- データ異常は否定（s2=長さ48・s3=132と正常、機械チェック通過）
- **アニメ実装が2系統ある**:
  - れんしゅう「みてみよう」＝**完了連鎖式**（構造上同時にならない）
  - **viewer.html と じぶん字典リプレイ＝絶対時刻の setTimeout 並べ** ← こちらが原因
- iOS Safari のタイマー合体（低電力・復帰時のクランプ）で**隣接タイマーが同一フレーム発火→2画同時開始**
- **恒久対策: 完了連鎖方式への統一（小規模）**。R6 で対応予定
- ⚠️神田さんへの確認事項: **どの画面で見えたか**（viewer か じぶん字典リプレイか）

### ★3. 部首データは代用不可と確定（最大のブロッカー）
- KANJIDIC2 の classical（康熙分類）は**漢検要覧と一致しない字があり、部首名も持たない**→代用不可
- 現実解: **『漢検要覧 2〜10級対応』（協会刊）から「部首記号・部首名・所属漢字」の3点テーブルを級ごとに段階整備**
- **書籍の入手は神田さん側の作業**
- データ側は Phase 1a のスキーマが `radical:{classical}` と分離済みなので `kanken` を並記するだけで受けられる
- 影響: 9級〜5級で一貫出題。**6級は部首そのものを記述させるため回避不能**

### ★4. 現行実装と新仕様の乖離
**出題（js/exam.js はほぼ全面作り直し）**
- 現行: 3タイプ×固定30問×一律5点をハードコード
- 新仕様: 級構成データファイル＋タイプのプール
- 影響: `buildExam()`・`EXAM_SPEC`（定数→データ化）・`scoreExam()`（一律5点前提）・index.html の `renderExamQ()` 3分岐→7タイプ以上のレンダラ群
- **そのまま残るもの**: 中断再開（毎問保存）・復習キュー・「もどってくる」・`readingForChoice` 等のかな整形（**読み2択の規則生成にそのまま使える**）・mulberry32/シャッフル・重複回避の考え方

**UI（index.html のマークアップ/CSSはほぼ全面書き直し）**
- 現行: 縦スマホ前提・5画面SPA・冷たい白＋濃紺
- 新方針: iPad横中心・学年二軸の階層（ホーム=みち型/学年でまなぶ/分野4入口/腕だめし/じぶん字典学年区画/学年設定）・積み木質感
- **css/tokens.css は拡張で済む**: 積み木パレットを追加しつつ、**ink系トークンは「中の道場」（書くマス・字・定着）でそのまま生きる**（デザイン言語の二層構造どおり）
- 新機構: 学年設定＋種類1文言の漢字/ルビ機械切替（既存の学年別データ＋KANJIDIC2読みで機械判定可。texts.js を静的文字列→ルビ対応の構造へ）

**データ**
- `sentences-g1.json` の「1文=1字」→**「1文に2〜3字」へのスキーマ変更が最大の改善余地**
- 正本 `tools/build_exam_g1.py` の改修＋既存164文の複数字化が必要
- **未完の照読と同じタイミングでやるのが効率的**

## 作り直しの分割案（R1〜R6）— 前セッションの提案・Chat承認済み

**方針「骨組み先行・エンジン並行」**: UI刷新と出題作り直しは腕だめし・れんしゅう画面で衝突するが、**出題エンジンのデータ/ロジック層は DOM 非依存**なので画面と独立に先行できる。完全統合（1フェーズ）は検証単位が大きすぎ、完全直列（UI全部→出題全部）は腕だめし画面を2度作ることになる。

| # | 内容 | 依存 |
|---|---|---|
| **R1** | 骨格: iPad横の画面地図・ナビ・学年設定・ルビ機構・積み木トークン（ホーム案Y＋学年でまなぶ。中身は現行機能へ橋渡し） | なし |
| **R2** | れんしゅう刷新（4入口B-2・にがて帯・「字を書く画面=道場」のモック確認をChatと） | R1 |
| **R3** | 出題エンジン層（級構成データ・かな手書き照合・筆順何番目・読み2択・かな補完・対のことばデータ・出題履歴・**1文複数字化＋照読**） | なし（**R1と並行可**） |
| **R4** | 腕だめし画面（案1）に R3 を接続＝81問本番＋分野別テスト | R2・R3 |
| **R5** | じぶん字典（案2・学年区画） | R1 |

- R3 は重いので「**R3a=データ形式と自動生成型／R3b=かな手書き入力**」に分割可
