# jibun-jiten Phase 3a 指示書（10級 問題データ生成：出題読みの選定＋文テーブル）

対象: Claude Code / 推奨モデル: Fable 5（大量コンテンツ生成の質が本体。台帳の昇格予定局面§2-2-2がここ）
前提: 台帳・決定事項リストを読むこと。Phase 2 検収済み。

---

## 位置づけ（台帳との差分・Chat決定）

Phase 3 を 3a / 3b に分割。**3a=問題データ、3b=ためす画面＋間違い還流**。
9級は Phase 3 から外す（小2の160字が未取込のため。10級で型を固めてから拡張）。
模擬テストは文がなければ1問も作れない。3a のゴールは **10級の読み・書取問題を支える文データが、品質チェック済みで揃うこと**。

## タスク1: 出題読みの選定

- 出力: `data/exam-readings-g1.json`
- 各字について、KANJIDIC2 全読みから小1相当の出題読みを音・訓それぞれ選ぶ（0個も可）
- 判断基準: **小1の生活で普通に使う読みだけ。迷ったら入れない**（狭く始めて後で足す方が安全。逆は出題済み問題の削除になる）
- 機械チェック: 選定読みがすべて kanji-g1.json の読み集合に含まれること・全80字にエントリがあること
- ⚠️この選定は判断を含むため全件が照読対象（タスク3）

## タスク2: 文テーブル生成

- 出力: `data/sentences-g1.json`（schemaVersion / sources 封筒は 1a の型を踏襲）
- レコード: `{ id, text, target: { char, reading, start, len }, use: ["yomi","kaki"] }`
  - 1文を読み問題・書取問題の両用にしてよい（同一テスト内での重複回避は 3b の責務）
- 分量: 1字につき2〜3文（80字で160〜240文）。全80字をカバー
- 品質規範（すべて必須）:
  - 文長8〜18文字・ひらがな主体・漢字表記は対象字のみ（他の語はひらがなで書く。既習判定の複雑さを持ち込まない）
  - 対象字の読みは、タスク1で選定した出題読みのいずれかであること
  - 小1の生活語彙・具体的な場面。固有名詞・商品名・キャラクター名なし
  - 場面の偏りに配慮（家族構成・性別役割・季節をばらす。全部「おかあさんと」にしない）
  - 既存の問題集・ドリル・教材を参照しない（著作権。自前生成のみ）
- 機械チェック: 文中に現れる漢字が target.char のみ / 文長範囲 / text の start..len 位置が対象表記と一致 / reading が選定読みに含まれる / 全80字にyomi用・kaki用が各1文以上

## タスク3: 照読の段取り（このフェーズの検収方式）

- 全データを1枚の確認ページ（sentences-check.html。字・出題読み・文・用途の一覧、装飾不要）に出力して push
- 停止して完了報告（選定読みの方針サマリ＋文のサンプル5件＋機械チェック結果）

## 検証（神田さんの作業）

1. 確認ページを開いて流し読み（10分程度を想定）
2. 違和感のある文・読みを番号で指摘 → Code が修正
3. 全部は読み切れなくてよい。気になったものだけで可（機械チェックが形式面は押さえている。ここで見てほしいのは「小1にとって自然か」だけ）

## 台帳更新（完了報告時に、Chat承認済みとして反映してよい）

- Phase 3 を 3a/3b に分割・9級は後続拡張へ（本指示書の決定）
- ★教訓追記: ①Node の純ロジック自己テストは IndexedDB 実挙動を検出できない → ブラウザ実走を必須にする ②保存を持つアプリの実機確認は通常ウィンドウ＋バージョン表示で（シークレットは保存が消える）

## やらないこと

模擬テスト画面 / 採点・合否 / 筆順画数問題の生成（既存データから 3b で自動生成）/ 9級・小2の字 / 三字熟語などの上位級分野 / イラスト・装飾
