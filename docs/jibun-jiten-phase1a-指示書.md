# jibun-jiten Phase 1a 指示書（データ基盤：スキーマ＋KanjiVG取込＋80字ビューア）

対象: Claude Code / 想定モデル: Opus 4.8（正解の決まっている構造化作業のため）
前提: 開始時に docs/jibun-jiten-handoff.md と決定事項リストを読むこと。

---

## 位置づけ（台帳との差分）

台帳の「Phase 1」は Chat 側の決定で 1a / 1b に分割済み。本指示書はその 1a。

- **1a（本書）**: スキーマ確定・KanjiVG/KANJIDIC2 取込・10級80字ビューア
- **1b（別指示書・未作成）**: 照合エンジンv1。本書ではやらない
- 筆跡保存（IndexedDB）は Phase 2 に送り済み。本書ではやらない

1a のゴールはただ1つ: **スキーマが実データ80字で破綻しないと確認すること**。ここが緑なら残り946字への拡張は安全と判断できる。

## タスク1: データソースの取得と抽出

1. KanjiVG を取得（GitHub: KanjiVG/kanjivg）。SVGファイル名は Unicode コードポイント小文字5桁hex（例: 字 = 05b57.svg）
2. KANJIDIC2 を取得（EDRDG 配布の kanjidic2.xml）。読み（音・訓）・画数・部首番号のソースとして使う
   - ⚠️取得時に同梱のライセンス条件を必ず確認し、正確な文言・バージョンを NOTICE に転記すること（CC BY-SA 系のはずだが、原文を正とする）
3. 対象は小1配当の80字（下記）。機械チェック: 文字数がちょうど80であること、全字に KanjiVG ファイルが存在することを確認

```
一右雨円王音下火花貝学気九休玉金空月犬見五口校左三山子四糸字耳七車手十出女小上森人水正生青夕石赤千川先早草足村大男竹中虫町天田土二日入年白八百文木本名目立力林六
```

⚠️このリストは Chat 側の知識による。文科省の学年別漢字配当表と1字でも食い違いを見つけたら、修正せず停止して報告

## タスク2: スキーマ確定と字テーブル生成

- 出力: `data/kanji-g1.json`（学年ごとにファイルを分ける。全学年1ファイルにしない）
- スキーマ（決定事項リスト§6の具体化）:

```json
{
  "char": "字",
  "grade": 1,
  "kenLevel": 10,
  "strokeCount": 6,
  "radical": { "classical": 39 },
  "readings": { "on": ["ジ"], "kun": ["あざ"] },
  "strokes": [ { "type": "㇔", "path": "M..." } ]
}
```

- 設計判断（Chat 決定済み・変更する場合は停止して相談）:
  - strokes は KanjiVG の path(d) と画種(kvg:type) をそのまま保持する。座標サンプリング列への変換は 1b（照合仕様が決まってから）。理由: 照合の設計前に形式を固定すると手戻りになる
  - readings は KANJIDIC2 の全読みを入れる。「小1で習う読み」への絞り込みは本書の範囲外（未解決課題として台帳に記録する）
  - radical は KANJIDIC2 の classical 番号を仮置き。漢検要覧準拠の部首は既知の未解決（Phase 4 ブロッカー）。フィールドを classical と分けてあるのは後で kanken を並記するため
- 変換はスクリプト化して `tools/` に残す（手作業で JSON を書かない）。理由: 1026字拡張時に同じスクリプトを再実行するため。再現可能性が本体
- 機械チェック: 全80字で ①strokeCount と strokes 配列長が一致 ②readings が空でない ③JSON が valid

## タスク3: 80字ビューア

- 出力: `viewer.html`（既存の index.html＝書き味テストはそのまま残す。1b で使う）
- 内容: 80字のグリッド → タップで詳細（字・画数・読み・筆順アニメ）
- 筆順アニメは KanjiVG の path を順に stroke-dashoffset で描く定番手法でよい
- ページにバージョン表示（★反映確認の教訓。push 後1〜2分・シークレットウィンドウ）
- 装飾は最小限。これは検収用の道具であり、子ども向け UI は Phase 2

## タスク4: ライセンス表記

- リポジトリ直下に `NOTICE.md`: KanjiVG（CC BY-SA 3.0）と KANJIDIC2（取得時に確認した正確な条件）の帰属表記
- viewer.html のフッターにも1行でデータ出典を表示

## 検証（神田さんの作業。手順を提示して停止）

1. 公開URLの viewer.html を実機で開き、バージョン確認
2. 5字だけ確認: 右と左（1画目が違う有名ペア。右はノが先・左は一が先になっているか）、山・糸・田（画数・読み・アニメの自然さ）
3. 気になる字があれば字を指定して報告

全80字の悉皆チェックは求めない（機械チェック済みのため）。
