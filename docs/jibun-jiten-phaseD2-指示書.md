# jibun-jiten Phase D2 指示書（全画面デザイン適用・一括）

対象: Claude Code / 推奨モデル: Fable 5（アプリ全体の質を決める最大の仕上げフェーズ）
前提: 台帳・決定事項リスト・jibun-jiten-デザイン言語.md・home-mock.html を必ず読むこと。既存実装 v0.7。

---

## このフェーズの位置づけ

当初からの旗＝「無料でも有料級・親子で"すごい"」。核（署名演出「定着」）は D1 で実機承認済み。
だが大半の画面は素っ気ない実務UIのまま（けん玉画面のような白背景＋青枠）。
本フェーズで全画面にデザイン言語を一括適用し、アプリ全体を"すごい"水準へ引き上げる。これがUI仕上げの本体。

## 絶対条件（違反したら失敗）

1. **機能・データ・配線を一切変えない。見た目だけを被せる。**
   - 定着演出 / IndexedDB保存(v2)と3枚ルール / テスト中断再開 / 復習キュー / なんでかな / matcher / 出題ロジック / 自己テスト — すべて挙動を保つ
   - 変更は CSS とマークアップ構造とごく表層の描画に限る。ロジック(js/*.js の判定・保存・状態遷移)には原則触れない
2. **書き味を落とさない**（Phase 0 canvas の入力経路を触らない）
3. 全画面で デザイン言語のトークン(8色・書体・原則)に準拠。home-mock.html の質感を全画面の基準にする
4. 既存の自己テスト3種(matcher/store/exam)を全通過のまま保つ。回帰したら停止

## デザイントークンの一元化（最初にやる）

- デザイン言語の8色・書体・角丸・余白を CSS カスタムプロパティ(:root)の1ファイルに定義し、全画面がそれを参照する形にする
- home-mock.html のトークンを正として移植（--paper #F7F8FA / --ink #1B2A4A / --grow #3A9D6B 等）
- 以後の画面はこのトークンだけで組む。個別画面でハードコード色を書かない

## 画面ごとの適用方針

### 1. ホーム

- home-mock.html の設計を本体に組み込む（モック→実装）。主役は「これまで書いた字の壁」、濃紺インクの「きょうの5もじ」カード、進み具合の帯、下部にためす／じぶんじてんの2タイル
- 壁の習得済み字は本人の筆跡(定着後の描画)、未習得は ghost の破線。数はデータに追従

### 2. じぶん字典（独自性が最も濃い・作品集）

- 80マスに、習得済み=本人の定着筆跡を作品として。未習得=ghost。今日ぶん=ふちに新緑の点
- タップ詳細: はじめて/ベスト/さいきん の3枚を額装するように静かに並べる＋順再生リプレイ
- 「じぶんの てで かいた ○もじ」の署名的フッター
- ここは最も時間をかけてよい。定着した字が一番美しく見える画面にする

### 3. れんしゅう各画面（子どもが最も長く見る）

- よみ＋筆順アニメ / なぞり / そらがき の各画面をトークンで統一
- 手本アニメは ink 色・ガイドは ghost。書くマスは home-mock と同じ紙の質感
- 「おしい！」「できた！」の文言表示をデザイン言語で（うるさくしない・新緑は点まで）
- ③そらがき合格の定着演出は D1 のものをそのまま維持（触らない）

### 4. ためす（テスト画面）

- よみ・かくすうの3択ボタン、かきの手書きマスをトークンで統一
- テスト中は静かに（デザインも演出過多にしない。測る場の緊張感を壊さない・救済表示は出さない仕様を維持）
- 進行表示(○もんめ/30もん)・中断ボタンを上品に

### 5. 結果画面＋なんでかな

- 点数/150 は数字に性格のある書体で作品のように（デザイン言語の数字方針）
- 合格/不合格/満点の3態の文言はデザイン言語で（不合格も前向き・禁止語なし維持）
- なんでかなの重ね表示(あお=手本/くろ=自分/問題画の強調)は機能を保ちつつ配色をトークンへ。「もういちど かく」の逃げ道を維持

## タスク分割（一括だが内部は段階的に・各段階で自己テスト確認）

1. トークンCSS一元化＋ホーム組み込み → 中間報告(続行可)
2. じぶん字典
3. れんしゅう各画面（定着は保持）
4. ためす＋結果＋なんでかな
5. 全画面の通し確認・バージョン更新(v0.8想定) → 完了で停止・報告

各段階の終わりに自己テスト3種を実行し、回帰ゼロを確認しながら進む。

## 検証（神田さんの作業。手順提示して停止）

通常ウィンドウ・実機(スマホとiPad)で:

1. ホーム→れんしゅう5字→じぶん字典→ためす→結果→なんでかな、と一周する
2. 見るのは2点: (a) 全画面が一貫して"すごい"か・ちぐはぐな画面がないか (b) 機能が全部これまでどおり動くか（定着・保存・中断再開・復習・なんでかな・書き味）
3. 画面ごとに気になれば指摘 → 個別調整

## 台帳更新（完了報告時）

- 「この節目メモを台帳に反映」（現在地・宿題・分岐・順序の教訓の一式。Chatが /tmp に整理済みの内容）
